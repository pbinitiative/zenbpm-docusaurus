# How to Start a Process Instance

This guide explains how to start a new process instance from a deployed BPMN process definition.

## Prerequisites

- ZenBPM server is running and accessible
- A process definition has been [deployed](./deploy-process-definition)

## Start a Process Instance

Use the REST API to start a new instance of a deployed process definition:

<ApiOperation id="api" pointer="#/paths/~1process-instances/post" example={true} />

### Request body

| Field | Type | Required | Description |
|---|---|---|---|
| `processDefinitionKey` | string | ✅ | Key of the deployed process definition |
| `variables` | object | ❌ | Initial process variables passed to the instance |
| `businessKey` | string | ❌ | External identifier to correlate the instance with a business entity |

### Example

```json
{
  "processDefinitionKey": "order-process",
  "businessKey": "ORDER-12345",
  "variables": {
    "customerId": "C-001",
    "orderAmount": 1500,
    "priority": "high"
  }
}
```

The response contains the `processInstanceKey` you will need to query or interact with the instance later.
